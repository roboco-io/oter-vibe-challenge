# 면접(발표) 안내

## 형식

- **온라인** 진행
- **30분 발표**: 바이브 코딩(Claude Code 등)으로 준비한 덱을 활용해 과제 진행 과정과 결과를 발표합니다.
- **30분 Q&A**: 발표 내용을 바탕으로 질의응답을 진행합니다.

## 질의 중심 주제

- **ADR 근거**: 왜 그 스택을 선택했는가, 다른 대안은 무엇을 고려했고 왜 배제했는가.
- **E2E 동등성 설계**: 마이그레이션본과 재구현본의 동등성을 어떤 기준과 방식으로 검증했는가.
- **회고 궤적**: 일주일간 Retrobot 회고와 tokenhabit 리포트에서 어떤 변화·개선이 있었는가, 그 이유는 무엇인가.

## 준비 팁

- 완성도를 자랑하기보다, **의사결정 과정과 트레이드오프**를 설명할 수 있도록 준비하세요.
- 실패하거나 포기한 시도가 있었다면 숨기지 말고, 왜 그런 선택을 했는지 이야기하세요. 이 과제는 완성도보다 설계 의도와 성장을 봅니다.
